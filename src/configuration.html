Tailwind CSS Configuration

Using Tailwind Without Installation

To test Tailwind in the browser without requiring a build step, use the Play CDN. The Play CDN is not meant for production; rather, it is exclusively meant for development. 

<script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4"></script>

Install Tailwind CLI

The recommended way to use Tailwind CSS in a project is via the CLI. Install it using npm: 

npm install -D tailwindcss

Install Tailwind IntelliSense for VS Code

To get class name autocompletion, install the official Tailwind CSS IntelliSense extension from the VS Code marketplace.

Download Tailwind IntelliSense

For the extension to work you have to create otherwise optional file tailwind.config.js

Minimal configuration:

module.exports = {
  content: [],
  theme: {
    extend: {},
  },
  plugins: [],
}

Enable Class Sorting with Prettier

Add the following plugin to sort Tailwind classes automatically when formatting code with Prettier: 

npm install -D prettier prettier-plugin-tailwindcss

Then, create or update your .prettierrc file: 

{
  "plugins": ["prettier-plugin-tailwindcss"]
}

Set Up PostCSS

Simply install PostCSS for Tailwind with: 

npm install -D tailwindcss postcss autoprefixer

You can add this to package.json to build easily with npm run build:css, assuming files' paths as follows: 

"scripts": {
  "build:css": "npx @tailwindcss/cli -i ./src/css/style.css -o ./src/css/main.css --minify",
  "watch:css": "npx @tailwindcss/cli -i ./src/css/style.css -o ./src/css/main.css --watch"
}

Note that you'll need to add '@import "tailwindcss";' to your style.css file.
